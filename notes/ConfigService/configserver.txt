Q) What code modifications required if Config Server IP/PORT is modified?
A)
  Create bootstrap.properties file inside MS# (every MS) 
  and add key=val of Config Server ex:
    spring.cloud.config.uri=http://localhost:9900

--src/main/resources
  |--application.properties
  |--bootstrap.properties

------------------------------------------------------
Why like that?

=> Our Project is Child Projects
=> It is connected to Parent project
=> Our Project has only dependency : Config Client
   code exist at parent project

=> For every project there will be properties data

   parent project(code) <---application.properties
         
   child project(code) <--- application.properties

=>** We want to give input to parent project using child project
   then we should define bootstrap.properties
   which is loaded before our application.properties
   on project-startup(while executing config client)

*** When we start MS# first it will fetch data from config server
    using config client, then MS# execution is started.
    So, if we add key inside our application.proerties 
    which is loaded after config client.
